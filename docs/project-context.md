# 儿童英语测评项目上下文

## 项目定位

这是一个儿童英语测评系统，分为两端：

- 老师端：登录后管理卷子、员工、答题记录
- 学生端：通过分享码进入指定卷子，填写信息后答题

当前技术栈：

- 前端：Vue 3、Vue Router、Ant Design Vue
- 后端：Node.js、Express、mysql2
- 数据库：MySQL

## 当前页面结构

### 老师端

- `#/papers`
  - 卷子列表
  - 搜索 / 筛选
  - 查看本卷答题情况
  - 新建卷子 / 编辑卷子
- `#/papers/new`
  - 卷子基础信息
  - 互动奖励配置
  - 题型配置
- `#/answers?paperId=...`
  - 当前卷子的答题记录
  - 支持按学生检索
- `#/users`
  - 员工列表
  - 重置密码 / 启用禁用
- `#/users/new`
  - 独立新增员工页

### 学生端

- `#/join`
  - 通过 6 位分享码进入
- `#/paper/:shareCode`
  - 学生填写姓名 / 手机号 / 年龄 / 年级
  - 答题流程
  - 结果页
  - 开屏动画 / 完成动画 / 转盘抽奖

## 已支持题型

当前题型包括：

- 听音选图
- 听音做指令
- 看图选词
- 拖拽组句
- 单词跟读
- 拼写填空
- 听题口答
- 听音选字母
- 图文跟读
- 图片连线

## 当前重要产品规则

### 老师端

- 老师端使用独立后台样式
- 新建员工已拆到独立页面 `#/users/new`
- 新建卷子默认是空白卷子
- 卷子总分必须配置到 100 分才能保存
- 已有答题记录的卷子不能再直接编辑，只能复制后再改

### 学生端

- 学生端不暴露老师端配置入口
- 学生信息必填项：
  - 姓名
  - 手机号
  - 年龄
  - 年级 / 班级
- 默认不显示错误提示
- 点击“开始闯关”后才显示未填写字段的提示

## 动画与抽奖

### 当前动画 / 奖励机制

每张卷子可单独配置：

- 是否开启开屏动画
- 是否开启完成动画
- 是否开启转盘抽奖
- 奖品列表
- 概率权重

学生端流程为：

1. 进入卷子
2. 如果卷子开启开屏动画，则先显示开屏
3. 学生提交答卷
4. 如果开启完成动画，则先播放完成动画
5. 完成动画后进入转盘
6. 学生点击“开始抽奖”时才真正按概率抽奖

### 抽奖约束

- 每张卷子单独配置
- 每次答卷记录（submission）只能抽一次
- 抽奖结果会写回这次 submission

## 图片处理策略

老师端配置图片类题型时，当前采用：

- 老师选择本地图片上传
- 前端读取为 data URL
- 数据直接保存在卷子配置里

这样第一版不需要单独的文件服务器，学生端可以直接展示图片。

## 语音播放说明

学生端“听示范 / 播放问题”等使用浏览器 `speechSynthesis`：

- 优先使用英文 voice
- 如果当前设备没有英文 voice，则回退到系统默认 voice

已知现象：

- 某些 Windows 环境只有中文 voice
- 当前逻辑已做 fallback，不再强绑 `en-US`

## 启动方式

在项目根目录执行：

```bash
npm install
npm run api
npm run serve
```

默认：

- 前端开发服务：`http://127.0.0.1:8080`
- 后端 API：`http://127.0.0.1:3001`
