# 2026-04-13 音频反馈动画设计

## 目标
为儿童英语测评中的“录音题”和“听力播放题”增加更适合 3-6 岁儿童理解的状态反馈动画，让孩子不用依赖文字也能直觉知道“现在在听”还是“现在轮到我说”。

## 结论
采用 `方案 B：小熊老师陪伴` 作为统一视觉角色。

## 设计原则
- 面向 3-6 岁：优先使用角色动作和大图形信号，而不是复杂文字说明。
- 陪伴感为主：像卡通老师或小伙伴，不做过强的游戏化刺激。
- 关键状态明显：开始录音和录音中要比普通静态页更明确，避免孩子不知道何时开口。
- 跨题型统一：听示范、听力播放、开始录音、录音中、录音完成都围绕同一个“小熊老师”展开。

## 统一角色设定
- 角色：小熊老师
- 颜色：暖黄、奶油白、天蓝
- 表情：友好、微笑、不过度夸张
- 动作语言：
  - 听的时候：歪头、侧耳、旁边有轻微声波或音符
  - 说的时候：正面朝向孩子、麦克风亮起、底部有温和波形
  - 完成时：点头、微笑、轻量成功反馈

## 状态设计
### 1. 听示范 / 正在播放
- 小熊微微歪头，做出“认真听”的姿态。
- 小熊耳边出现 2-3 条柔和声波线或音符。
- 按钮状态：播放按钮亮起，但不抢主视觉。
- 提示文案：例如“我先读一遍给你听”。
- 目标：孩子知道当前不需要说，先听。

### 2. 开始录音
- 点击“开始录音”后，小熊从侧耳状态切到正面状态。
- 麦克风按钮由普通状态切到高亮状态，并带一次轻微放大反馈。
- 可以出现非常短的 1 次呼吸光圈，提示录音已经开始。
- 目标：孩子知道“轮到我说了”。

### 3. 录音中
- 小熊保持正面陪伴姿态。
- 底部出现柔和波形，随录音状态起伏，但不要做高频闪烁。
- 麦克风按钮持续发光。
- 小熊可以有轻微眨眼或点头，增强陪伴感。
- 提示文案：例如“我在认真听你说哦”。
- 目标：让孩子持续感知系统正在听，而不是卡住了。

### 4. 录音完成
- 波形减弱并收起。
- 小熊微笑或点头。
- 如果后续有评分结果，可以在结果出现前给一个轻量过渡，比如“听到了，正在看看你读得怎么样”。
- 目标：平滑过渡到评分结果，避免突兀。

## 在现有项目中的落点
当前最适合先改造的题型是：
- `D:\aa-workplace\EnglishQuestion\src\components\questions\ReadAloud.vue`

后续可复用到：
- 听力播放类题型（如 `ListenChooseImage.vue`、`ListenFollowInstruction.vue`）

## 组件改造建议
### ReadAloud.vue
- 用小熊老师替换现有相对抽象的录音视觉中心。
- 保留当前分数环和波形，但将其风格改得更柔和、更低龄。
- 将状态拆成：
  - idle
  - demo_playing
  - recording
  - scored
- 不要求第一版就接真实录音动画数据，可以先基于状态切换做视觉反馈。

### 听力题复用
- 播放听力时，统一让小熊进入 `listening` 状态。
- 这样孩子跨题型会形成稳定心智：
  - 小熊侧耳 = 先听
  - 小熊正面 + 麦克风亮 = 轮到我说

## 动效强度控制
- 不做快速闪烁和高频跳动。
- 动画周期建议在 1.2s - 2.4s 之间。
- 波形高度起伏控制在柔和范围，不做像 KTV 那样强烈的柱状条。
- 所有成功反馈都用轻反馈，不用大幅弹跳。

## 第一版实现范围
- 先实现录音题的 4 个状态视觉切换。
- 听力题先只加“侧耳听”的播放状态反馈。
- 第一版不引入复杂骨骼动画，只用 CSS 动画和状态切换实现。

## 风险与约束
- 当前项目已有较儿童化的 UI，新增角色必须和现有配色协调，不能像另一个产品拼进来。
- 如果小熊插画资源过大，会继续放大当前构建图片体积 warning，需要后续做压缩。
- 第一版重点是“状态清晰”，不是追求复杂动画。

## 验收标准
- 小朋友能明显分辨“正在听”和“轮到我说”。
- 录音题点击开始录音后，界面有即时明显反馈。
- 听力题播放时，界面不是静止的。
- 风格与当前儿童英语测评产品一致，不显得过于抽象或成人化。
