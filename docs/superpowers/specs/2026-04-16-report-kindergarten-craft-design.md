# 幼儿园手工风报告页改版设计

## 背景

当前报告页已经具备完整的数据承载能力，但视觉语言仍明显偏后台统计页：

- 顶部文案像系统结果页，而不是儿童成长反馈
- KPI、能力条、雷达图、评语之间缺少统一的儿童化主题
- 页面对 3-6 岁儿童和家长来说不够亲和，奖励感不足

本次改版目标不是改变报告数据结构，而是在保留现有导出能力、现有字段和现有页面结构的前提下，把报告页调整为“幼儿园手工风”。

## 目标

- 让报告页更像“老师整理的成长记录板”，而不是管理后台
- 保留现有四块核心内容：顶部摘要、KPI、能力展示、教师评语
- 提高导出图片和导出 HTML 时的观感
- 不新增后端字段，不改变评分逻辑，不改变报告生成逻辑

## 非目标

- 不改动报告数据模型
- 不改动提交答卷和生成 report 的 store 逻辑
- 不新增复杂交互动画
- 不调整老师端页面风格

## 设计方向

### 整体气质

页面采用“幼儿园成长记录板”风格：

- 底色像纸张和公告板
- 卡片像便签、贴纸和手工卡
- 局部使用胶带、印章、星星等轻装饰
- 文字表达更偏鼓励和陪伴

视觉关键词：

- 奶油色底
- 柔和糖果色
- 大圆角
- 轻阴影
- 轻微手工拼贴感

### 信息层级

1. 先给孩子和家长“今天成长得不错”的正向感受
2. 再展示数字结果
3. 再展示能力表现
4. 最后用老师评语收尾

## 页面结构

### 1. 顶部成长头图区域

保留现有 hero 区，但改为成长记录语义：

- 主标题：`今日英语成长记录`
- 副标题：展示学生姓名、年龄、学校、联系方式
- 操作按钮保留：重新作答、查看作答记录、导出图片、下载 HTML
- chip 行改为更儿童化的“贴纸徽章”样式

设计要求：

- 标题区域采用暖色背景或浅色渐变
- 加入不影响导出的装饰，如小星星、云朵、胶带角标
- 行为按钮仍需清晰可点，但视觉上融入整体配色

### 2. KPI 区

保留当前 4 个指标：

- 综合得分
- 总分
- 完成率
- 题目数量

改为 4 张不同颜色的小卡：

- 黄：综合得分
- 蓝：完成率
- 绿：题目数量
- 粉：总分或成长徽章类信息

设计要求：

- 每张卡都是独立圆角卡片
- 数字更大、更醒目
- 标签文案更口语化，但保留识别性

### 3. 能力展示区

保留两部分：

- 能力条形图
- 能力雷达图

处理方式：

- 条形图主视觉优先，做成“蜡笔色表现条”
- 雷达图作为辅助说明，降低后台图表感

设计要求：

- 条形图轨道更柔和
- 填充色使用浅蓝、薄荷绿、桃粉等儿童配色
- 雷达图外圈、轴线和文字做弱化，不抢主视觉

### 4. 教师评语区

将当前评语区强化为页面情感核心：

- 外观像老师写给孩子的一张留言便签
- 可以加小图标或印章感标题
- 保持当前 report comments 的展示逻辑不变

设计要求：

- 卡片面积适当放大
- 行高增加，增强可读性
- 如果评语较短，也能撑起完整视觉块

## 文案调整

在不改变数据含义的前提下，页面展示文案调整为更亲和的表达：

- `测评报告已生成` -> `今日英语成长记录`
- `能力分布` -> `听说读表现`
- `教师评语` -> `老师想对你说`

KPI 文案允许做轻量优化，但必须保持可理解性，避免过于口语化导致家长不清楚指标含义。

## 颜色与样式规范

建议主色：

- 页面底色：`#FFFBF3`
- 奶油黄：`#FFE9A8`
- 天空蓝：`#BFE7FF`
- 薄荷绿：`#CDEFD6`
- 草莓粉：`#FFD6E7`
- 辅助文字：`#6B7280`
- 深标题色：`#5B4A36`

组件规范：

- 大卡片圆角：24px
- 小卡片圆角：20px
- 阴影：轻柔，不做重投影
- 边框：浅棕或浅蓝灰，不用纯黑

## 实现范围

### 代码层面

主要修改：

- `src/views/ReportView.vue`
- `src/views/PaperView.vue` 中报告态的同类结构
- `src/styles.css` 中报告页相关样式

### 数据层面

不修改：

- `state.report`
- ability map 计算逻辑
- report comments 解析逻辑
- 导出逻辑

## 风险与兼容

- 报告页支持导出图片，装饰元素不能依赖复杂外链资源
- 页面仍需兼容电脑和 iPad
- 文案和装饰增加后不能导致单屏明显溢出
- `PaperView.vue` 和 `ReportView.vue` 的报告态风格需要保持一致，避免两个入口样式分裂

## 验证方案

- 检查报告页浏览显示是否完成改版
- 检查导出 PNG 是否正常
- 检查下载 HTML 是否仍可用
- 检查电脑宽屏与 iPad 尺寸下是否保持可读
- 检查无评语、能力项较少时页面是否仍稳定

## 结论

推荐以“幼儿园手工风”为主线完成本轮改版，并优先复用当前报告结构，只重做文案、卡片层级和装饰样式。这样既能明显提升儿童感，也能控制改动范围和回归风险。
