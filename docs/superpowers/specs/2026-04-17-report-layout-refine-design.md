# 报告页版式收缩设计

## 背景

在上一轮“幼儿园手工风”报告页改版后，页面整体气质已经更贴近儿童成长记录，但当前信息重心仍然不够聚焦：

- `老师想对你说` 占用了过高的版面
- `今日收获` 这块信息和顶部贴纸、KPI 信息重复
- `听说读表现` 与 `能力小雷达` 两块真正承载核心结果的内容，视觉面积还不够大

本轮只做版式收缩，不改变数据结构、不增加新模块。

## 目标

- 删掉重复的 `今日收获` 大卡片
- 将 `老师想对你说` 从大卡片压缩成上方一条窄横条
- 把腾出来的大块空间优先分配给：
  - `听说读表现`
  - `能力小雷达`
- 让页面主次更明确：上方摘要，下面两块核心结果

## 选定方案

采用方案 1：

- 保留顶部标题、学生信息、导出按钮、贴纸行
- 保留 KPI 四卡
- `老师想对你说` 改成一条细横条，放在 KPI 下方、双主卡上方
- `今日收获` 整块移除
- `听说读表现` 与 `能力小雷达` 成为下方主视觉，尽量等高放大

## 页面结构

新的布局顺序如下：

1. 顶部 Hero 区
2. 贴纸行
3. KPI 四卡
4. 细横条评语区
5. 双主卡区
   - 左：听说读表现
   - 右：能力小雷达

## 具体调整

### 1. 评语区

从原来的大面积黄卡，调整为一条横向成长便签：

- 高度缩到单行或双行文案
- 保留标题 `老师想对你说`
- 视觉仍保留便签感，但弱化边框和留白
- 不再占据页面下半区

### 2. 今日收获区

整块删除：

- 不再显示 `今日收获`
- 顶部贴纸中已有 `今天收到了...`，足够承担奖励反馈
- KPI 也已覆盖总分、完成率、题量等摘要信息

### 3. 双主卡区

重点扩大：

- `听说读表现` 卡片增加高度
- `能力小雷达` 卡片增加高度
- 两块尽量同高，形成页面主体
- 让用户进入报告页后，第一眼看到能力结果而不是次要摘要

## 非目标

- 不改动评分逻辑
- 不改动导出逻辑
- 不新增新的后端字段
- 不改单独的报告数据结构

## 涉及文件

- `src/components/shared/StudentCraftReport.vue`
- `src/views/PaperView.vue`
- `src/views/ReportView.vue`（如果仍保留独立报告入口）
- `src/styles.css`

## 验证点

- 页面中不再出现 `今日收获` 大卡
- `老师想对你说` 改成窄横条
- `听说读表现` 和 `能力小雷达` 两块明显变大
- 导出 PNG / HTML 时版式仍稳定
- iPad 和桌面端下双主卡不挤压、不溢出
